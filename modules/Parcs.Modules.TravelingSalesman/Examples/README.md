# Тестові файли для TSP модуля

Ця папка містить тестові файли з містами для перевірки роботи Traveling Salesman Problem модуля.

## Формат файлів

Всі файли мають формат:
```
# Формат: ID X Y
# Кожен рядок представляє одне місто
0 100.50 200.30
1 150.75 180.45
2 200.25 250.80
...
```

Де:
- `ID` - унікальний ідентифікатор міста (ціле число)
- `X` - X-координата міста (число з плаваючою комою)
- `Y` - Y-координата міста (число з плаваючою комою)

## Доступні файли

### cities_500.txt
- **Кількість міст**: 500
- **Розподіл**: Випадковий
- **Координати**: 0-1000
- **Призначення**: Тестування середнього масштабу

### cities_1000_sample.txt
- **Кількість міст**: 50 (зразок)
- **Розподіл**: Випадковий
- **Координати**: 0-1000
- **Призначення**: Зразок для 1000 міст

## Генерація великих файлів

Для створення файлів з більшою кількістю міст використовуйте `GenerateLargeCities.cs`:

```bash
# Збірка та запуск генератора
dotnet build
dotnet run --project . --no-build -- GenerateLargeCities
```

### Доступні патерни генерації:

1. **Випадковий розподіл** - міста розподілені випадково по всій площині
2. **Кластеризований розподіл** - міста згруповані в кластери
3. **Сітковий розподіл** - міста розташовані у вигляді сітки

### Розміри файлів:

- `cities_1000.txt` - 1000 міст
- `cities_2000.txt` - 2000 міст  
- `cities_5000.txt` - 5000 міст

## Використання в TSP модулі

Для завантаження міст з файлу встановіть в `ModuleOptions`:

```csharp
var options = new ModuleOptions
{
    LoadFromFile = true,
    InputFile = "cities_500.txt", // або інший файл
    // ... інші параметри
};
```

## Приклади використання

### Послідовний модуль
```csharp
// Модуль автоматично завантажить міста з файлу
var module = new SequentialMainModule();
// Результат буде збережено у файли, вказані в options
```

### Паралельний модуль
```csharp
// Паралельна обробка з розподілом по точках
var module = new ParallelMainModule();
// Кожна точка отримає копію міст та виконає локальний GA
```

## Продуктивність

Очікувана продуктивність на різних розмірах:

| Кількість міст | Послідовний час | Паралельний (4 точки) | Прискорення |
|----------------|-----------------|----------------------|-------------|
| 500           | ~2-5 сек        | ~1-2 сек             | 2-3x        |
| 1000          | ~10-20 сек      | ~3-6 сек             | 3-4x        |
| 2000          | ~40-80 сек      | ~10-20 сек           | 4-5x        |
| 5000          | ~200-400 сек    | ~50-100 сек          | 4-5x        |

*Час залежить від параметрів генетичного алгоритму та обладнання*

## Налаштування параметрів

Для великих задач рекомендується:

```csharp
var options = new ModuleOptions
{
    PopulationSize = 2000,      // Більша популяція
    Generations = 200,          // Більше поколінь
    MutationRate = 0.005,       // Менша мутація
    CrossoverRate = 0.9,        // Більший кросовер
    PointsNumber = 8,           // Більше точок для паралелізації
    SaveResults = true
};
```

## Вимоги до системи

- **RAM**: Мінімум 4GB для 1000+ міст
- **CPU**: Багатоядерний процесор для паралельної обробки
- **Диск**: ~1MB на 1000 міст для збереження результатів
